New JavaScript Features (A must know for all JavaScript developers)

Introduction

Welcome to the only webpage that covers new JavaScript features from 2016 till date. I put it all together from various sources to keep JavaScript developers (who are not aware of these new features) up to date with the development of JavaScript, so they can get familiar with them and start using them as soon as all major browsers fully support them. I hope this truly helps, and if it does, gladly upvote the code.

As some of you may know, the standards of JavaScript and some other scripting languages (e.g. JScript) is derived from another programming language. That language is known as ECMAScript. So any feature added to ECMAScript is also a JavaScript feature and they are practically considered the same language. This webpage covers all the major features added yearly to ECMAScript from the year 2016 up to July 2020. If you don't know ECMAScript 2015 (formerly known as ES6), I suggest you check it out here. None of them have a large feature release compared to ECMAScript 2015.

Before you start consuming the page's content with your brain, I should tell you that this page is pretty long and it will take much of your time. But I assure you that it is worth it.

ECMAScript 2016

ECMAScript 2016 introduced very few features compared to ES 6:

  • Array.prototype.includes()
  • Exponentiation Operator

Array.prototype.includes()

Array.prototype.includes() checks the array for the value passed as an argument. It returns true if the array contains the value, otherwise, it returns false.

nums = [0, 1, 2, 3];
if (nums.includes(2)) { ...console.log('Array contains value'); }

Before, we needed to use Array.prototype.indexOf() to check if the given array contains an element or not.

let nums = [1, 2, 3, 4];
if (nums.indexOf(2) !== -1) { ...console.log('Array contains value'); }

Array.prototype.includes() handles NaN better than Array.prototype.indexOf(). If the array contains NaN, then indexOf() does not return a correct index while searching for NaN.

Array.prototype.includes() returns the correct value when searching for NaN.

const array = [1, 2, 3, 4, NaN];
Instead of if (array.indexOf(3) >= 3) { ...console.log(true); }
Use if (array.includes(3)) { ...console.log(true); }
Note that indexOf() doesn't work for searching NaN array.includes(NaN); true array.indexOf(NaN); -1

Exponentiation Operator

JavaScript already supports many arithmetic operators like +, -, * and more. ECMAScript 2016 introduced the exponentiation operator, **. It has the same purpose as Math.pow(). It returns the first argument raised to the power of the second argument.

You can also perform x **= y for directly assigning the value to the variable x.

let base = 3; let exponent = 4;
You can either do this let result = base ** exponent; console.log(result); 81
Or this base **= exponent console.log(base) 81

ECMAScript 2017

ECMAScript 2017 introduced features to support asynchronous programming, shared memory (don't worry about that for now) and other stuffs:

  • Object.entries
  • Object.values
  • String padding
  • Object.getOwnPropertyDescriptors
  • Trailing commas in function parameter lists and calls
  • Async functions

Object.entries

When a JavaScript datatype with key value pairs is passed to the entries method, the return value is a two dimensional array of keys and values. This makes it very simple to do things like using objects in loops or converting objects into Maps.

Using objects in loops:

const cars = { BMW: 3, Tesla: 2, Toyota: 1 };
ES 5.1 Instead of extracting keys and then again looping Object.keys(cars).forEach(function(key) { ...console.log("key: " + key + " value: " + cars[key]); });
ECMAScript 2017 (ES 8) Use Object.entries for ([key, value] of Object.entries(cars)) { ...console.log(`key: ${key} value: ${value}`); }

Converting objects into map:

const cars = { 'BMW': 3, 'Tesla': 2, 'Toyota': 1 };
ES2015 Instead of.. Get object keys, then add each key to a map in a loop const map1 = new Map() Object.keys(cars).forEach(key => { ...map1.set(key, cars[key]); });
console.log(map1) Map { 'BMW' => 3, 'Tesla' => 2, 'Toyota' => 1 }

ES2017 and onwards
Use const map = new Map(Object.entries(cars));
console.log(map) Map { 'BMW' => 3, 'Tesla' => 2, 'Toyota' => 1 }

If you don't know what a map object is in JavaScript, click here for more info.

Object.values

Object.values works similar to Object.keys, but instead of returning only the keys, it returns a single dimension array of the values.

const cars = { 'BMW': 3, 'Tesla': 2, 'Toyota': 1 };
ES2015 Instead of.. const values = Object.keys(cars).map(key => { cars[key] }); console.log(values); [3, 2, 1]
ES2017 and onwards
Use const values = Object.values(cars); console.log(values); [3, 2, 1]

String Padding

Two methods padStart and padEnd has been added to String to support the padding at the beginning and the end of the string. If you want to format a list of numbers for display purposes, they're your best bet.

The padStart method:

const nums = [0, 1, 12, 123, 1234];
nums.map(value => { ...value = value.toString().padStart(4, 0); Prepend zero until length is 4 ...console.log(value); });
Prints out the following 0000
0001
0012
0123
1234

The padEnd method:

const nums = [0, 1, 12, 123, 1234];
nums.map(value => { ...value = value.toString().padEnd(4, 0); Append zero until length is 4 ...console.log(value); });
Prints out the following 0000
1000
1200
1230
1234

padStart and padEnd with Object.entries:

const cars = { 'BMW': '11', 'Tesla': '6', 'Lanborghini': '0' }
Object.entries(cars).map(([car, count]) => { ...padEnd appends ' -' until the name becomes 20 characters ...padStart prepends '0' until the count becomes 3 characters. ...console.log(`${car.padEnd(20, ' -')} Count: ${count.padStart(3, 0)}`); });
BMW - - - - - - - - Count: 011 Tesla - - - - - - - Count: 006 Lanborghini - - - - Count: 000

They are very helpful if, for example, you want to create a digital clock, and want to prepend zero to all numbers involved as long as they are less than 10.

You can try appending or prepending emojis to a string with these methods. You'll be surprised with the results 😉.

Object.getOwnPropertyDescriptors()

This method takes in as a parameter an object for which to get all own property descriptors. Then it returns all the details (including getter and setter methods) for all the properties of a given object.

const obj = { ...property1: 42 };
const discriptors = Object.getOwnPropertyDiscriptors(obj);
console.log(discriptors.property1.writable); true
console.log(discriptors.property1.value); 42

Note that this feature is supported in the current versions of all major browsers except IE

Trailing Commas

This feature makes rearranging items simpler, because you don’t have to add commas if the last item changes its position. If you ask me, I don't see the point in it.

Some examples:

function about(name, age,) { ...return `${name} is ${age} years old.` }; console.log(about('Ade', 12)); Ade is 12 years old

Async/Await

The async keyword, when put in front of a function, makes the function asynchronous and return a Promise. If you are wondering what callback hell is, I've included a video tutorial from allthingsjavascript. Check it out below:

This feature relies on ES6/ES2015 promises. If you don't understand or know about promises, click here for more info.

A trivial example:

When the async keyword is put in front of a function declaration, it returns a promise async function hello() { ...return "Hello"; }; hello();
You can use it with any type of function let hello = async function() { ...return "Hello" }; hello();
let hello = async () => { ...return "Hello" }; hello();
They all do the same thing.
